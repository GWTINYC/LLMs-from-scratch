# 关于如何最大化从技术书籍中受益的建议

以下是我以前当读者询问如何从我关于“从零构建大语言模型”的书籍中获得最大收获时分享的一些笔记。

我自己阅读技术书籍时也遵循类似的方法。这并非通用的准则，但可以作为一个有用的起点。

对于这一本特定的书，我强烈建议**按顺序阅读**，因为每一章都建立在前一章的基础上。每一章，我建议按照以下步骤进行：

&nbsp;
### 1) 第一遍阅读（离线）

我建议先从头到尾阅读该章节，**不要编写任何代码**。
这第一遍阅读的目标是先掌握**全局图景 (Big Picture)**。

理想情况下，我建议远离电脑阅读该章节。实体书效果很好，但没有干扰（没有浏览器、社交媒体或电子邮件）的数字设备也可以。

就我个人而言，我在纸质书和电子墨水平板上都阅读。虽然自 2018 年以来我一直使用电子墨水平板，并且总是尝试更多地使用电子墨水阅读，但我仍然注意到实体书能帮助我更好地集中注意力。这也是为什么我有时会因为想详细了解而打印一些具有挑战性的研究论文。

我的建议是让第一遍阅读成为一次简短、专注的 20 分钟阅读，尽量减少干扰，不要过度思考或纠结于细节。

标记或注释令人困惑或有趣的部分是可以的，但在现阶段我不建议去查资料。我只建议阅读，先不要运行任何代码。这第一遍是为了理解大局。

&nbsp;
### 2) 第二遍阅读（带代码）

在第二遍阅读时，我建议**手动输入并运行**章节中的代码。复制粘贴代码很诱人，因为重新输入工作量很大，但当我阅读其他技术书籍时，手动输入通常能帮助我多思考一下代码（而不是仅仅扫一眼）。

如果我得到的结果与书中的不同，我会检查书的 GitHub 仓库并尝试那里的代码。如果仍然得到不同的结果，我会尝试看看是否是因为不同的包版本、随机种子、CPU/CUDA 等原因。如果之后仍然搞不清楚，向作者提问也不失为一个好主意（通过书的论坛、公共 GitHub 仓库的 issues 或讨论区，最后手段是发邮件）。

&nbsp;
### 3) 练习 (Exercises)

在第二遍阅读、重新输入并运行代码之后，通常是尝试**练习**的好时机。这对巩固理解或以半结构化的方式修补问题非常有帮助。如果练习太具挑战性，看解决方案是可以的。但是，我仍然建议首先认真尝试一下。

&nbsp;
### 4) 回顾笔记并进一步探索

现在，在阅读完章节、运行代码并完成练习之后，我建议回顾前两遍阅读时的**高亮和注释**，看看是否还有不清楚的地方。

这也是查阅额外参考资料或进行快速搜索以澄清任何仍感觉未解决的问题的好时机。即使一切都讲得通，阅读更多关于感兴趣的主题也不是坏主意。

在这个阶段，将有用的见解、代码片段等写下来或转移到你喜欢的笔记应用中也是有意义的。

&nbsp;
### 5) 在项目中使用这些想法

前面的步骤都是关于吸收知识。现在，看看你是否可以在**自己的项目**中使用某一章的某些方面。或者也许可以使用书中的代码作为起点构建一个小项目。为了获得灵感，可以查看**Bonus Materials (额外材料)**，这些基本上是我为了满足自己的好奇心而做的迷你项目。

例如，在阅读了多头注意力机制 (Multi-head Attention) 并实现了 LLM 之后，你可能会想知道具有分组查询注意力 (Grouped-Query Attention) 的模型表现如何，或者 RMSNorm 与 LayerNorm 真的有多大区别。诸如此类。

也可能有一些较小的方面可以在你自己的项目中有用。例如，有时微小的细节最终会变得有用，比如测试显式调用 `torch.mps.manual_seed(seed)` 是否比仅使用 `torch.manual_seed(seed)` 有任何不同。

不过，最终我也想以某种方式使用这些知识。这可能涉及使用该章节的主要概念，但也涉及我在此过程中学到的一些小知识点，例如，即使是琐碎的事情，比如显式调用 `torch.mps.manual_seed(seed)` 而不仅仅是 `torch.manual_seed(seed)` 是否真的会对我的项目产生影响。

&nbsp;
### 额外的想法

当然，以上均非金科玉律。如果主题总体上非常熟悉或简单，而我主要是为了获取后面章节的一些信息而阅读这本书，那么**略读**一章是可以的（以免浪费时间）。

另外，对于没有任何代码的章节（例如介绍性的第 1 章），跳过与代码相关的步骤当然是有意义的。

无论如何，希望这对你有用。祝你阅读和学习愉快！
